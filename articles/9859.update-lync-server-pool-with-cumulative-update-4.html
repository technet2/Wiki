Update Front End

Stop the  World Wide Web Publishing Service  net stop w3svc


 


Then stop all services Lync Server server. Run the Lync Server Management Shell cmdlet: Stop-CsWindowsService


 
 
Copy the files to the Front End , and run the package LyncServerUpdateInstaller.exe
 

Select Install Updates .


The wizard updates the server roles installed on the server
 

And finalize the upgrade wizard
 

Rerun the cmdlet  Stop-CsWindowsService: 

Stop-CsWindowsService

And start the service  World Wide Web Publishing Service  

net start  w3svc 

 

Updates to the Back End must be installed separately. After finishing the wizard run the cmdlet Lync Server Management Shell . CsDatabase-Install-Update-ConfiguredDatabases-SqlServerFqdn <FQDN server Back-End> UseDefaultSqlPaths 

 

 

The cmdlet updates instances of SQL Server on the server


With the updates installed start the services Lync Server, return to Lync Server Management Shell and run the cmdlet  Start-CsWindowsService  

Start-CsWindowsService 
 

Follow the steps to upgrade the other servers in your pool to keep the whole structure updated.

Group Policy to Update Client Lync

Copy the update files to a shared folder, the folder's NTFS permissions add the Domain Computers group with the right reading. 
 

Create a script file with the 64-bit client update. The script should contain the command line: msiexec / update \ \ HM01 \ fs \a name="Group_Policy_para_Atualiza_ccedil_atilde_o_do_Lync_Client">

Group Policy to Update Client Lync
Copy the update files to a shared folder, the folder's NTFS permissions add the Domain Computers group with the right reading. 

 

Create a GPO Update Lync Client , expand the Properties Computer Configuration, Policies, Windows Settings and select Scripts . Open the properties of the script Startup 
 

Click Add and select the upgrade script
 
 
 
 

Click OK to finish the configuration
 

Apply the GPO in the OU station
 
 

As the stations are restarted, the client update is applied.
